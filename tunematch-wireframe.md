# TuneMatch — 와이어프레임 문서
> 유튜브 취향 궁합 분석 서비스  
> 플랫폼: 웹 (모바일 우선)  
> 작성일: 2025

---

## 전체 화면 구조

```
랜딩 페이지
  └─ OAuth 연동 화면 (A)
       └─ 링크 전송 화면
            └─ 대기 화면 (A)
            └─ 링크 접속 화면 (B)
                 └─ OAuth 연동 화면 (B)
                      └─ 결과 화면
                           ├─ 공유 카드 화면
                           └─ 결과 저장 화면
```

## 유저 Journey 요약

```
Phase 1 · DISCOVERY
  SNS 공유 카드 발견 / 검색 / 직접 접속
  └─ 랜딩 페이지 도착

Phase 2 · ONBOARDING (A 기준)
  랜딩 페이지
  └─ "궁합 분석 시작" CTA
  └─ Google OAuth 연동
  └─ 고유 링크 생성
  └─ 카카오/문자로 상대방에게 링크 전송

Phase 3 · WAITING
  A: 대기 페이지 (이탈 허용)
  └─ 링크 재전송 버튼
  └─ 페이지 떠나도 OK

  B: 링크 접속
  └─ 서비스 간단 소개
  └─ Google OAuth 연동

  B 연동 완료 →
  └─ A에게 이메일 알림 발송
  └─ 분석 자동 시작

Phase 4 · RESULT
  궁합 점수 카운트업 애니메이션
  └─ 총점 공개
  └─ 세부 항목 순서대로 공개
       ├ 채널 겹침 분석
       ├ 카테고리 DNA (레이더 차트)
       ├ 취향 유형 코멘트
       └ 함께 볼 채널 추천
  └─ 광고 노출 (이 타이밍에만)

Phase 5 · SHARE & RETURN
  공유 카드 생성
  └─ 이미지 저장 / 카카오 공유 / 링크 복사

  결과 저장 옵션 (선택)
  └─ YES → Google 계정으로 간편 가입
  └─ NO  → 그냥 종료

  다른 친구와 또 하기 → Phase 2 루프
```

## 궁합 점수 산정 방식

```
총점 100점 = 3가지 지표 조합

① 채널 겹침률          30점
   두 사람이 공통으로 구독한 채널 수 / 전체 구독 채널 합
   (Jaccard 유사도)

② 카테고리 취향 유사도  50점
   각자의 카테고리 비중 벡터를 코사인 유사도로 비교
   채널이 하나도 안 겹쳐도 취향 결이 같으면 높은 점수

③ 시청 패턴 유사도      20점
   좋아요 영상 기반 간접 추정
   (시청 기록은 MVP 제외)
```

## 카테고리 분류 (8개)

```
🎮 엔터 / 게임       게임, 예능, 드라마 리뷰
📚 지식 / 교육       강의, 다큐, 사이언스
😂 유머 / 밈         코미디, 숏폼, 반응 영상
💪 라이프스타일      운동, 뷰티, 브이로그, 여행
🎵 음악 / 퍼포먼스   뮤직비디오, 커버, 공연
📰 뉴스 / 시사       정치, 경제, 사회
🍳 음식 / 요리       먹방, 레시피, 맛집
💻 테크 / 비즈니스   IT, 스타트업, 재테크
```

- 재방문 명분 (3개월 후 재분석) 을 저장 시점에 심어둠

## 핵심 설계 원칙 요약

```
1. 마찰 최소화
   OAuth 연동 → 링크 전송까지 3탭 이내

2. 이탈 허용 설계
   대기 중 이탈 OK → 이메일 알림으로 복귀 유도

3. 프라이버시 선제 해소
   모든 연동 화면에 수집 항목 명시

4. 바이럴 루프
   결과 → 공유 카드 → 신규 유입 → 결과 → ...

5. 광고 최소 노출
   결과 확인 후 단 1회만 → 몰입감 유지

6. 선택적 로그인
   결과 확인 후 저장 욕구 생겼을 때 가입 유도
```
